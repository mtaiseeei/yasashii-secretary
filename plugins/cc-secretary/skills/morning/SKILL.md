# 朝ブリーフィング（morning）

毎朝、その日の段取りを秘書のほうから届けるためのスキル。中身の作り方は「今日やること」（daily）が土台で、
**朝向けの体裁（今日の要点・最初の一歩）** に整えて返す。定時実行を設定していれば毎朝自動で届き、
**設定していなくても「おはよう」で同じものが手動で得られる**（自動化はあくまで上乗せ）。

ユーザーに話しかける前に、必ず `${CLAUDE_PLUGIN_ROOT}/rules/plain-language.md`（言葉づかいルール）を読むこと。
報告は3行以内（やったこと／結果／次に何が起きるか）。一般に通じる技術用語はそのまま使い、馴染みの薄い語だけ初出で簡潔に補足する。

## ブリーフの作り方（daily を土台に・二重実装しない）

まず `${CLAUDE_PLUGIN_ROOT}/skills/daily/SKILL.md` を段階ロードし、そのステップ1〜3に従って材料を集める。
**根拠ルール・同期しない不変条件・未接続フォールバックは daily と同一**（このスキルで緩めない）。

- 各項目に必ず根拠（**サービス名＋リンク/ID＋日付**）を付ける。原文にない予定・締切を推測で作らない。
- 外部データの本文（メール本文・予定の詳細）を**ローカルに保存しない**。キャッシュファイルも作らない。ローカルに残してよいのは TODO と根拠参照まで。
- コネクタが**未接続でも壊さない**: ローカル TODO（`secretary/inbox/todo.md`）だけで要点を伝え、接続案内（`${CLAUDE_PLUGIN_ROOT}/skills/setup-google/SKILL.md`）へやさしく橋渡しする。

### 朝向けの体裁（ブロック構成）

集めた材料を、次の**ブロックの積み重ね**で届ける。ブロックは上から順に出し、
新しい項目（例: 長く待っている返信の警告など）が将来増えたら、**ブロックとして間に差し込める**構成を保つ。

1. **あいさつ＋今日の日付**（1行。呼び方は `secretary/memory/preferences.md` に従う）
2. **今日の要点**（2〜3件に絞る。予定×TODO の突き合わせ結果。各項目に根拠つき）
3. **最初の一歩**（まず何から始めるかを一言。迷わせない）

例:

> おはようございます、村山さん。7月15日（火）の朝ブリーフです。
> 今日の要点は3つ: 10時の打合せ（Googleカレンダー / 2026-07-15）、請求書の送付（TODO）、資料への返信（Gmail / 2026-07-15）。
> まずは10時の打合せ資料の確認から始めるのがおすすめです。「打合せの準備を手伝って」と言ってください。

長い一覧は既定では出さない。「もっと詳しく」と聞かれたときだけ広げる。

## 毎朝自動で届くようにする（定時実行のセットアップ案内）

「毎朝届くようにしたい」「自動でブリーフがほしい」と言われたら、この節の手順で案内する。
自動実行は**ホスト（Claude 側）の定時実行機能に薄く乗る**。プラグイン側に常駐プロセスや独自の cron（定時実行の仕組み）は作らない。

はじめに、方式が3つあること・**設定しなくても「おはよう」で同じものが得られる**ことを伝える。

| 方式 | 位置づけ | ひとこと |
|---|---|---|
| Claude デスクトップアプリの scheduled tasks（スケジュール済みタスク） | **本命** | パソコンの中で動き、プラグインと秘書ディレクトリをそのまま使える |
| Routines（ルーティン。claude.ai のクラウド定時実行） | 代替 | 環境によってはプラグインやローカルのファイルに届かないことがある（その旨を正直に伝える） |
| 手動（「おはよう」と話しかける） | いつでも使える | 設定不要。定時実行が未設定・不調でも同じブリーフが得られる |

### ステップ0: しおりを書いてから始める（中断に備える）

定時実行の設定は、設定画面への移動や Claude の再起動を挟むことがある。**設定に進む前に**再起動しおりを書く:

```
${CLAUDE_PLUGIN_ROOT}/skills/memory-care/scripts/memory-tools.sh resume-write <secretary> \
  "朝ブリーフの定時実行セットアップ" "設定画面で定時実行を登録→テンプレート貼り付け" "毎朝何時に届けるか"
```

再起動しても、次に秘書を呼べばここから続きを案内できる。設定が終わったら `resume-clear <secretary>` でしおりを閉じる。

### ステップ1: 貼り付ける文面（テンプレート）を渡す

`${CLAUDE_PLUGIN_ROOT}/templates/prompts/morning-brief.md` を読み、その中の枠内（貼り付け用の文面）を
ユーザーにそのまま提示する。**文面は書き換え不要・それ単体で意図が伝わる**ようになっている。

### ステップ2: 本命 — デスクトップアプリの scheduled tasks に登録する

画面の名前は今後変わることがあるため、迷ったら「**設定を開く → 定時実行（スケジュール）の登録を探す → テンプレートを貼り付ける**」という型で探せばよい、と先に伝える。執筆時点の具体的な場所:

1. Claude デスクトップアプリの **設定（Settings）** を開く。
2. **スケジュール済みタスク（Scheduled tasks）** の項目を開き、**新規作成（New task / Create）** を押す。
3. **実行時刻（Schedule / Time）** を毎朝の希望時刻（例: 朝8時）にする。
4. **プロンプト（Prompt / Instructions）** の欄に、ステップ1の文面を貼り付けて保存（Save）する。

### ステップ3: 代替 — Routines（クラウドの定時実行）

デスクトップアプリの scheduled tasks が使えない場合は、claude.ai の **Routines（ルーティン）** に同じ文面を登録する。
ただし正直に添える: Routines はクラウド側で動くため、**環境によってはプラグインやローカルの秘書ディレクトリに届かない**ことがある。
その場合でも「おはよう」と話しかければ、手元で同じブリーフが得られる。

### ステップ4: 確認と完了報告（3行型）

登録できたら「明日の朝、届いたか教えてください」と一言添え、しおりを閉じて3行で報告する。例:

> 朝ブリーフの定時実行を登録しました（毎朝8時・scheduled tasks）。
> 貼り付けた文面はテンプレートのままなので、時刻はあとから設定画面で変えられます。
> 次は明日の朝、ブリーフが届きます。届かなければ「おはよう」と話しかけてください。同じものをすぐお出しします。

## やらないこと（このスキルの範囲）

- プラグイン側に常駐プロセス・独自の定時実行の仕組みを作らない（登録先はホストの機能だけ）。
- メールの送信・削除などの不可逆な操作はしない。
- 外部データの本文をローカルに保存しない（daily と同一の不変条件）。
