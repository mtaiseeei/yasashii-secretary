# 夕方の振り返り（evening）

一日の終わりに、3分ほどの短い対話で今日を締める。やることは4つだけ:
**(1) 完了した TODO の消し込み → (2) 未完了の明日への繰越 → (3) ひとことの振り返り → (4) 節目コミット**。

ユーザーに話しかける前に、必ず `${CLAUDE_PLUGIN_ROOT}/rules/plain-language.md`（言葉づかいルール）を読むこと。
報告は3行以内。一般に通じる技術用語はそのまま使い、馴染みの薄い語だけ初出で簡潔に補足する。

**大事な保護**: 消し込み・繰越は、**何をどうするかを先に見せて、ユーザーの OK をもらってから**実行する
（記憶保護と同系の保護）。**無確認の一括削除をしない**。TODO の操作は必ず決定的シーム
`${CLAUDE_PLUGIN_ROOT}/scripts/workspace-tools.sh` を使う（封じ込めガードを通り、秘書ディレクトリの外は触らない）。

## ステップ1: 今日の TODO を並べる（完了確認）

まず今日の TODO を見せて、どれが終わったかを聞く。

1. `${CLAUDE_PLUGIN_ROOT}/scripts/workspace-tools.sh todo-list <secretary>` で一覧を取り、
   **「- [ ]」の行に上から 1, 2, 3… と番号を付けて**提示する（この番号が消し込み・繰越の指定に使われる）。
2. 「今日終わったものはどれですか？」と聞く。例:

> おつかれさまです。今日の TODO はこの3件でした。
> 1. 請求書を送る　2. 打合せ資料の確認　3. 資料への返信
> 終わったものの番号を教えてください（例:「1と2」）。全部まだなら「まだ」で大丈夫です。

TODO がまだ1件も無ければ、消し込み・繰越は飛ばしてステップ3（ひとこと振り返り）へ進む。

## ステップ2: 消し込みと繰越（確認してから実行）

### 消し込み（完了した項目）

終わったと言われた項目について、**消す内容を見せて OK をもらってから**、1件ずつ消し込む:

```
${CLAUDE_PLUGIN_ROOT}/scripts/workspace-tools.sh todo-done <secretary> <番号> --confirm
```

- `--confirm` を付けるのは**ユーザーの OK をもらった後だけ**。もらう前に付けない。
- `--confirm` なしで実行すると、消す対象を表示して中断する（安全側）。この表示を確認の材料に使ってよい。
- 複数終わっていても**一括で消さず1件ずつ**。番号は消すたびに詰まるので、**大きい番号から順に**消すと取り違えない。

### 繰越（未完了の項目）

残った項目は「明日へ繰り越しますか？ それとももう不要ですか？」と聞く。繰り越すものは、OK をもらってから:

```
${CLAUDE_PLUGIN_ROOT}/scripts/workspace-tools.sh todo-carry <secretary> <番号> <明日のYYYY-MM-DD> --confirm
```

繰り越した項目は消えずに「（繰越: 日付）」が付いて残り、明日の朝ブリーフでそのまま拾われる。
「もう不要」と言われた項目だけ、消す内容を確認のうえ `todo-done` で消す。

## ステップ3: ひとことの振り返り（記憶に残すのは価値のあるものだけ）

「今日、決めたことや気づいたことはありますか？（なければ『なし』で大丈夫です）」と一言だけ聞く。

- **残す価値があるもの**（確定した決定・好みの変化）だけを記憶に反映する:
  - 決定: `${CLAUDE_PLUGIN_ROOT}/skills/memory-care/scripts/memory-tools.sh remember-decision <secretary> <今日のYYYY-MM-DD> "<決定の内容>"`
  - 好みの変化: `memory-care` スキル（`${CLAUDE_PLUGIN_ROOT}/skills/memory-care/SKILL.md`）の手順で `preferences.md` を更新
- 「なし」なら何も書かない。**毎日機械的に記憶ファイルを量産しない**（日記帳ではなく、記憶の目次を汚さない）。
- 相対日付（「明日」「来週」）は絶対日付（YYYY-MM-DD）に直して記録する。

## ステップ4: 節目コミットと締めの報告

一日の締めとして、秘書ディレクトリの変更をローカルにコミットする（**push はしない**）:

```
${CLAUDE_PLUGIN_ROOT}/skills/memory-care/scripts/memory-tools.sh commit <secretary> "一日の締め（TODO の消し込みと明日への繰越）"
```

メッセージは日本語で、その日の内容が一目でわかるように。最後に3行型で締める
（**今日できたこと／明日へ送ること／次に何が起きるか**）。例:

> 今日は2件完了しました（請求書の送付・打合せ資料の確認）。
> 「資料への返信」は明日（2026-07-16）へ繰り越し、区切りをローカルに記録しました（インターネットには送っていません）。
> 次は明日の朝ブリーフでこの繰越を最初にお知らせします。おつかれさまでした。

## やらないこと（このスキルの範囲）

- 無確認の一括削除・一括変更（1件ずつ、確認してから）。
- 外部データ（メール・予定）への操作。ここで触るのはローカルの TODO と記憶だけ。
- push・リモート設定（コミットはローカルのみ）。
