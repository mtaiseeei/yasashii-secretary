# できること（機能の使い方）

いま使える機能の一覧です。呼び方の例のように話しかけると、秘書が対応する機能を読み込んで進めます。

## 記憶（覚える・守る・前回の続き）

- 呼び方の例: 「〇〇を覚えておいて」「これは決まり」「〇〇を消して」「振り返って」「前回の続き」
- できること:
  - 決めたこと・好みを `secretary/memory/` に記録し、目次（`MEMORY.md`）を最新に保ちます。
  - **守る**: 空の内容で上書きしない・消す前に「何を消すか」を確認する。うっかり消えないようにします。
  - **前回の続き**: 作業を中断しても、次に呼ぶと続きから案内します（再起動しおり）。

## 今日やること

- 呼び方の例: 「今日やること」「今日の予定」「TODO」「段取り」
- できること: つないだサービスの予定と、ローカルのTODO（`secretary/inbox/todo.md`）を突き合わせて、その日の要点を**根拠つき3行**で返します。
  - 各項目に「サービス名＋リンク/ID＋日付」の根拠を付けます。推測で予定を作りません。
  - 外部データの**全文はローカルに保存しません**（都度参照）。
  - まだ何もつないでいなくても壊れません。「先に Google をつなぎますか？」と案内します。

## 外部サービスにつなぐ

- Google（Gmail・カレンダー・ドライブ）／Microsoft 365（Outlook・OneDrive・Teams）／Notion（任意）。
- くわしくは [外部サービスにつなぐ（コネクタ）](connectors.md) を見てください。

## 接続の状態を診断する

- 呼び方の例: 「繋がってる？」「接続の調子」「診断して」「どれが使える？」
- できること: どのサービスが繋がっていて、どれがまだかを一覧で返します。未接続のものは、繋ぎ方の案内へ橋渡しします。

## 成果物を残す

- 「〇〇をまとめて」「企画書にして」などで作った文書は、`secretary/docs/YYYY/MM/YYYY-MM-DD_タイトル.md` の形で保存されます。
- 日付・見出し・目印（タグ）が付くので、あとから探しやすくなります。保存の節目でローカルにコミットします（push はしません）。

## 開発の入口（作って）

- 呼び方の例: 「〇〇なアプリ／ツールを作って」「これを実装して」「自動で開発を進めて」
- できること: 「計画→実装→検証」のループ（やさしいハーネス）に接続し、**いまどこにいるか**を見せながら開発を進めます。
  - 小さな1行直しだけなら、大げさにせず普通に対応します。

## まだできないこと（今後）

- 国内チャット（Chatwork / LINE 等）は対象外です（公式のリモート接続がまだ無いため）。今後の検討対象です。
