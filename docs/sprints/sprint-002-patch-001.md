# Sprint 002 Patch 001 — 記憶ツールの封じ込めハードニング

- Type: micro
- Base sprint: sprint-002（合格済み）
- Phase: P1（秘書コア）
- 主眼: 記憶操作の封じ込め（`secretary/memory/` の外に書き込みを届かせない）を、symlink 越えとエッジパスの2系統で塞ぐ。ユーザーから見た正常系の振る舞いは不変。

## なぜ Type: micro か

- 変更は封じ込め検査（memory-tools 相当の1ファイル）＋回帰の**負テスト追加**に閉じ、新しい機能・UI・正常系の振る舞いを増やさない。
- sprint-002 の既存回帰スイート（98 assert）がこの導線を既に守っており、負テストを足すだけ。
- したがって軽量評価（**機能完全性・動作安定性・回帰なし**のみ）で足りる。UX/独自性の再評価は不要。安全は本パッチの主題なので封じ込めの負テストで実証する。

## 課題（`docs/feedback/sprint-002.md` 再評価「ハードニング推奨」）

1. **symlink 越え書き込み**: `secretary/memory/` 内に外を指すシンボリックリンクがあると、封じ込め検査を通過した書き込みが実質的に外へ届く。
2. **`delete "."`（または空 rel）の偽装成功系**: エッジケースで、実際には対象がないのに「成功」と報告してしまう挙動。

## スコープ（含む）

1. **symlink 越えの遮断**: 書き込み/削除の対象パスを解決したとき、`secretary/memory/` の外へ抜ける symlink（対象自身・中間ディレクトリのいずれか）を検知したら操作を**拒否**する。実解決先が封じ込め境界の外なら書かない。
2. **エッジ rel の遮断**: `.`（カレント）、空文字、`memory` 直上や親方向へ抜ける相対参照（`..` 経由・境界脱出）を、**偽装成功させず拒否**する。「成功」報告を返さない。
3. **回帰の負テスト追加**: 上記2系統の拒否を assert する負テストを回帰スイートに追加（既存 98 assert は温存し、総数を増やす）。

## スコープ外

- 記憶ケアの正常系の振る舞い変更（索引・保護・しおり・振り返り・節目コミット）。
- 次スプリント（daily・コネクタ等）の機能。

## 受入基準（micro 軽量評価・検証可能な形）

Evaluator は以下を assert し、証跡（実行コマンドと結果）を `docs/feedback/sprint-002-patch-001.md` に残す。

1. **symlink 越え書き込みの拒否（機能完全性／安全, ゼロ許容）**: `secretary/memory/` 内に外を指す symlink を仕込んだ状態で書き込み/削除を試みると、**exit 3 で拒否**され、境界外の実ファイルが変更・作成・削除されていないこと。
2. **エッジ rel の非偽装（機能完全性／安全, ゼロ許容）**: `rel = "."` / 空文字 / 親方向へ脱出する相対参照に対する削除・書き込みが、**非ゼロで拒否**され、**「成功」と報告しない**こと（成功系の出力・exit 0 を返さない）。境界内の既存ファイルにも副作用がないこと。
3. **正常系の温存（動作安定性）**: 正当な `secretary/memory/` 内パスへの追加・更新・削除・索引追従・しおり・節目コミットが従来どおり成功する（sprint-002 の正常系が壊れない）。
4. **無回帰（ゼロ許容）**: 既存 98 assert が全パスし、新規の負テストが加わって総 assert 数が増えている。次スプリント機能の混入なし。push が発生していない（`git remote` 空）。

### exit コードの約束
- `3` = 封じ込め境界外（symlink 越えを含む）への書き込み/削除の拒否。
- 非ゼロ（`3` またはバリデーション用の別非ゼロ）= エッジ rel（`.`・空・境界脱出）の拒否。**いずれも exit 0 を返さない**。
- 実装上の細部（別コードの割り当て等）は Generator 裁量だが、上記の「境界越えは exit 3・エッジ rel は非ゼロで偽装成功しない」を満たすこと。

## sprint-002 契約との関係

- 本パッチは sprint-002 の**封じ込め・保護規則を強化**するもので、正常系の受入基準は不変で維持する。
- 受け入れられた封じ込め不変条件は `docs/spec/constraints.md`（安全）に恒久ルールとして反映済み（symlink 越え禁止・境界脱出の偽装成功禁止）。

## Generator への引き継ぎメモ

- 封じ込め判定は、対象パスの**実解決先**（symlink 解決後の正規化パス）が `secretary/memory/` 境界の内側にあるかで行う。境界文字列の前方一致だけに頼らない（`secretary/memory-evil/` のような接頭辞衝突に注意）。
- `.`・空 rel は「対象未指定」として早期に拒否し、削除の空振りを「成功」に丸めない。
- 負テストは、一時ディレクトリに `secretary/memory/` と外部ターゲットを作り、symlink を張って exit コードと境界外の不変を assert する形が扱いやすい。
- パス参照は `${CLAUDE_PLUGIN_ROOT}` 相対（patch-001 で確定）。push しない。
