# Sprint 003 — 今日やること・出力規約・Google 接続

- Phase: P1（秘書コア）
- 主眼: 秘書が外部データ（Google 系）を**根拠つき**で参照し、「今日やること」を3行で返せるようにする。成果物の出力規約を実運用で効かせ、Google コネクタ接続を公式リモート前提でガイドする。
- 依存: sprint-001（ワークスペース・ルーター・テンプレート）／sprint-001-patch-001（`${CLAUDE_PLUGIN_ROOT}` 参照）／sprint-002（記憶・`_resume.md`・節目コミット手順）

## なぜこのスプリントか

sprint-002 までで「記憶する秘書の家」ができた。sprint-003 は、その秘書が**外の予定・タスクを見て今日やることを整理し、成果物をきちんと残す**ようにする。非エンジニアにとっての価値は「毎朝これを聞けば今日の段取りが根拠つきでわかる」「作った文書が後から探せる形で残る」。コネクタは**同期せず都度参照**の原則を守る。

## スコープ（含む）

### 1. setup-google スキル（F09）
- `plugins/cc-secretary/skills/setup-google/SKILL.md`（frontmatter 有効・`name` 一意・ルーターから `${CLAUDE_PLUGIN_ROOT}` 相対で段階ロード）。
- **公式リモートコネクタ前提**の接続ガイド: Gmail / Google Calendar / Google Drive を、**Claude 側の設定画面からの公式コネクタ接続**（OAuth 自動）で繋ぐ手順。**Google Cloud Console の手作業（プロジェクト作成・API 有効化・認証情報発行）は案内しない**。
- **接続確認テスト**: 接続後に「繋がっているか」を軽く確認する手順（例: カレンダーの直近予定を1件読めるか）。失敗時は英語エラーをそのまま出さず「何が起きて・どうすれば直るか」に言い換える（「実エラーで原因確定してから案内」）。
- **再起動しおり連携**: コネクタ設定は Claude の再起動を挟むことがあるため、設定前に `secretary/memory/_resume.md`（sprint-002 のしおり）へ「今コネクタ接続の途中」という文脈を書き出し、再開時に続きから案内できるようにする。

### 2. daily スキル（F06）
- `plugins/cc-secretary/skills/daily/SKILL.md`（frontmatter 有効・段階ロード）。
- 接続済みコネクタから**予定・タスクを都度参照**し、ローカル TODO と突き合わせて、その日の要点を提示する。
- **ローカル TODO 源**: `secretary/` 配下のクイックキャプチャ／簡易 TODO（例: `inbox/` のメモ、または簡易 TODO リスト。正確な置き場は Generator 裁量だが **`secretary/` 配下・コミット対象**）。
- **根拠ルール**: 各項目に根拠（**サービス名＋リンク/ID＋日付**）を付す。原文にない事実を補完しない。矛盾は両方提示。
- **同期しない不変条件**: 外部データの**本文をローカルファイルに保存しない**。daily がローカルに残すのは「TODO・根拠参照（サービス名＋リンク＋日付）」まで。全文コピー・キャッシュファイルを作らない。
- **未接続時の親切なフォールバック**: コネクタ未接続でも daily は壊れず、「まず接続しますか？」と setup-google へ案内する（他機能を壊さない）。
- 報告は3行型（やったこと／結果／次に何が起きるか）。

### 3. 出力規約の実運用（F08）
- 成果物（企画書・調査まとめ等）を保存するとき、`secretary/docs/YYYY/MM/YYYY-MM-DD_<title>.md`、frontmatter 必須（`createdAt`＝`YYYY-MM-DD HH:mm` / `tags`）、1ファイル1トピック、見出しに固有名詞。
- 成果物作成は sprint-002 で一般化した**節目コミット手順**を再利用してローカルコミット（日本語メッセージ、push なし）。
- 実装はテンプレート／ヘルパー等の決定的なシームで、パス・frontmatter・命名を保証する（ドライラン検証可能に）。

### 4. ルーターへの組み込み
- `skills/secretary/SKILL.md` に、今日やること（今日／予定／TODO／段取り 等）・接続設定（Google／Gmail／カレンダー／接続 等）・成果物保存のモード判定を追加し、`${CLAUDE_PLUGIN_ROOT}` 相対で段階ロード。

## 受入基準（この契約は厚めに定義する）

コネクタは**実接続できない環境でも検証できる形**にする。Evaluator は `docs/spec/rubric.md` の方法で以下を assert し、証跡を `docs/feedback/sprint-003.md` に残す。

1. **スキル構文（C2, ゼロ許容）**: `setup-google` / `daily` の SKILL.md frontmatter が有効・`name` 一意。ルーター参照が `${CLAUDE_PLUGIN_ROOT}` 相対でデッドリンクなし。
2. **公式コネクタ前提の接続ガイド（C1, C4）**: setup-google のガイド文言に、**Google Cloud Console の手作業手順（プロジェクト作成・API 有効化・認証情報 JSON 等）が含まれない**こと（grep で不在を確認）。「Claude の設定画面から接続」に相当する導線と、接続確認テスト手順が存在する。
3. **英語エラーの言い換え（C4）**: setup-google が生の英語エラーをそのまま提示しない設計（言い換えの型・具体例が文言にある）ことを確認。
4. **根拠ルールの雛形（C3, C5）**: daily の出力雛形／手順に、各項目へ**サービス名＋リンク/ID＋日付**を付す規約が明示され、根拠なしで事実を断定しない旨が書かれている（雛形・文言検査）。
5. **同期層を作らない（C5, ゼロ許容の不変条件）**: daily の設計・雛形が外部データ本文のローカル保存を指示しない。ドライラン（サンプル TODO＋ダミー参照）で、daily がローカルに書くのは TODO・根拠参照までで、全文コピー/キャッシュファイルを作らないことを assert。
6. **未接続フォールバック（C1, C4）**: コネクタ未接続を模した状態で daily を起動しても失敗せず、setup-google への案内に落ちる（クラッシュ・スタックしない）。
7. **出力規約（C1, C3）**: 成果物保存のドライランで、`secretary/docs/YYYY/MM/YYYY-MM-DD_<title>.md` のパス・frontmatter（`createdAt`/`tags`）・1ファイル1トピックが満たされることを assert。保存の節目で日本語コミットが作られ **push されていない**（`git remote` 空）。
8. **再起動しおり連携（C3）**: setup-google が接続前に `_resume.md` へ文脈を書き、再開時に続きから案内できる導線があることを確認（sprint-002 のしおり機構を利用）。
9. **非エンジニア体験（C4）**: 接続ガイド・daily の出力が日常語＋具体例、報告3行型（「次に何が起きるか」を含む）。`rules/plain-language.md` を参照。
10. **安全・規律（C5, ゼロ許容）**: `~/workspace/agentic-harness` 書き込みなし。資格情報（トークン等）をワークスペースに保存/コミットしない。6規律・単段クレジット・封じ込め不変条件に反しない。
11. **無回帰（C6, ゼロ許容）**: sprint-001〜002（＋各 patch）の回帰スイートが全パス。本スプリントの assert を追加し実行コマンドを progress に記録。

### rubric 対応まとめ

## Generator への引き継ぎメモ

- **company の一次情報**（`~/workspace/inbox/company`、読むだけ・クレジット継承）: MCP 連携ガイドは SKILL.md **630-852 行**。ただしこれは **Google Cloud Console 手作業前提**なので、cc-secretary では**そのまま流用しない**。
  - **置き換え方針**: 「Claude 側設定画面からの公式リモートコネクタ接続（OAuth 自動）」に読み替える（DESIGN.md コネクタ設計参照）。company の手作業手順（プロジェクト作成・API 有効化・認証情報発行）は**削ぎ落とす**。継承するのは診断の型（実エラーで原因確定→日常語案内）と、接続確認テストの発想。
- **再起動しおりとの接続**: コネクタ設定は Claude 再起動を挟むことがあるため、sprint-002 の `_resume.md` プロトコルを使い「接続の途中」を記録→再開時に続きから案内する。
- **同期層を作らない不変条件**: daily は外部データを**都度参照**。全文をローカルに保存・キャッシュしない。残すのは TODO と根拠参照（サービス名＋リンク＋日付）まで。`10_sources` 型の層を作らない。
- ローカル TODO の置き場は最小限に（`secretary/` 配下・コミット対象）。スキーマを作り込みすぎない。
- パス参照は `${CLAUDE_PLUGIN_ROOT}` 相対。成果物・接続の節目コミットは push しない。
